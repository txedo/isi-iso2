A partir del diagrama de casos de uso de cada subsistema, se crean los diagramas de clases de análisis, para representar, a muy alto nivel, los requisitos funcionales de cada caso de uso y las clases de conocimiento involucradas.

A continuación, se listan y referencian los diagramas de clases de análisis de cada subsistema, correspondientes a cada uno de los casos de uso. En los diferentes diagrmas está comentado cuál es el flujo de eventos que se produce en cada uno de ellos.

Sin embargo, hay dos gestores que aparecen en la mayoría de los diagramas de clases de
análisis que merecen ser destacados: 

\begin{milista}
\item \textbf{Gestor de conexiones de bases de datos:} es una clase \textit{Controller} que servirá para mantener
sincronizadas las bases de datos utilizadas por el sistema, de manera que
cuando haya que hacer una inserción/modificación/eliminación, se haga en todas
las bases de datos, y si una operación falla se siga manteniendo la
sincronización, lo cual será imprescindible para el correcto funcionamiento
del sistema. En este caso, el gestor accederá a dos clases:
\begin{itemize}
	\item \textit{AgenteFrontend}, que es la interfaz (clase \textit{Boundary}) de acceso a la base de
	datos principal.
	\item \textit{ProxyServidorRespaldo}, que se comunica con el servidor de
	respaldo para poder utilizar la base de datos secundaria. Por ello, es un \textit{Boundary} para el servidor de respaldo.
\end{itemize}
	
\item \textbf{Gestor de conexiones de estado:} es una clase \textit{Controller} que servirá para que los mensajes
de estado generados por el servidor front-end (principalmente después de
que un usuario realice una operación) puedan ser procesados por todas las
clases que los quieran recibir. En este caso, cada mensaje será utilizado por
tres clases:
\begin{itemize}
	\item \textit{GestorConexionesBD}, para guardar el mensaje en la base
	de datos y así mantener un registro de todas las operaciones realizadas,
	como se pide en los requisitos. Esta clase es el gestor anterior.
	\item \textit{JFServidorFrontend}, para mostrar el mensaje en la ventana
	principal del servidor front-end, por lo que es una clase \textit{Boundary}.
	\item \textit{ProxyServidorRespaldo}, para enviar el mensaje al servidor
	de respaldo y que éste lo muestre en su ventana	principal. Com se vió, dicha clase es un \textit{Boundary}.
\end{itemize}
	
\end{milista}
