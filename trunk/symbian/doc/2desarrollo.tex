\section{Desarrollo de aplicaciones en Symbian}

\subsection{Versiones de Symbian}

En los siguientes apartados se van a comentar las diferentes versiones que existen del sistema operativo Symbian, destacando sus características más importantes.


\paragraph{Symbian OS 5.0, 5.1, 6.0 y 6.1}

Como ya se ha comentado en la sección \ref{queEsSymbian}, la versión 5.0 es la primera versión comercial de este sistema operativo. La siguiente versión, la 5.1, lanzada en el dispositivo \textit{Sony Ericsson R380}, comienza a soportar código UNICODE.

En el año 2001 se lanza la versión Symbian OS 6.0, utilizada en el teléfono móvil \textit{Nokia 9120 Communicator}. A partir de esta versión, este sistema operativo se encamina al desarrollo para teléfonos inteligentes \textit{smartphones}. En esta versión se incluye el soporte para conectividad Bluetooth y el soporte para la ejecución del conjunto de instrucciones de tipo ARM (\textit{Advanced RISC Machine}). Además, se intenta el desarrollo de interfaces de usuario separadas para los \textit{smartphones} y para los dispositivos más convencionales, obteniendo dos referentes: la interfaz \textit{Quartz} y la \textit{Crystal}. Sin embargo, aunque se había intentado un desarrollo común, se produjo una clara división, pasando Nokia a utilizar la primera de ellas y Sony Ericsson la segunda. \\
\indent Con esta separación, se comienza a hablar de la serie de dispositivos \textbf{Series 80 1st Edition}, que son los pertenecientes a Nokia, y la serie \textbf{UIQ 1.0 y UIQ 1.1}, pertenecientes a Sony Ericsson y Motorola, diferenciándose en el tipo de interfaz de usuario, aunque la plataforma y versión de Symbian utilizada es la misma en ambos.

En cuanto a la versión 6.1, se incorpora el soporte para una cámara VGA con resolución de 0.3 Mpx, utilizándose por primera vez en el \textit{Nokia 7650}, perteneciente a la serie \textbf{Serie 60 1st Edition}.


\paragraph{Symbian OS 7.0}

Esta versión se publica en el año 2003, añadiendo soporte para IPv6, para la tecnología EDGE (\textit{Enhanced Data Rates for GSM}) y para Java, siguiendo el estándar Java ME. Esta versión se utiliza con cualquier tipo de interfaz de usuario, incluyéndose en las series \textbf{Serie 80 2nd Edition}, \textbf{Serie 90 1st y 2nd Edition}, \textbf{Serie 60 2nd Edition}, \textbf{UIQ 2.0} y \textbf{UIQ 2.1}.

Por otra parte, ésta es la primera versión de Symbian con un problema de seguridad grave, ya que es infectada por el gusano llamado \textit{Cabir}, capaz de transmitirse a otros móviles con Symbian OS a través de Bluetooth.


\paragraph{Symbian OS 8.0 y 8.1}

La versión 8.0, lanzada en el año 2004, cuenta con dos kernel del sistema operativos diferentes, el EKA1 y el EKA2, aunque éste ultimo no se incorpora hasta la versión 8.1. Por tanto, la versión 8.1 es una revisión de la 8.0, para mejorar la compatibilidad con versiones anteriores y dar soporte a ambos kernels.

En cuanto a las nuevas características que ofrecen estas versiones, cabe destacar:

\begin{milista}
	\item Soporte para chips de memoria más compactos.
	\item Soporte para reconocimiento de voz.
	\item Mejora de la codificación de datos para aplicaciones multimedia (aplicaciones de audio y 3D).
	\item Incorporación de OpenGL y gráficos vectoriales.
	\item Se permite la administración remota del sistema.
	\item Soporte para conexión WCDMA (\textit{Wideband Code Division Multiple Access}).
	\item Se incorpora el protocolo para la comunicación a través de videollamadas.
	\item Soporte para tecnología 3G.
\end{milista}

Estas versiones se incluyen en las series \textbf{Series 60 2nd Edition 2.6} y \textbf{Series 60 2nd Edition 2.8} y se comercializa en teléfonos como el \textit{Nokia N70}, que tanto éxito tuvo en el mercado.


\paragraph{Symbian OS 9.0 - 9.5}

La versión 9.0 de este sistema operativo, lanzada en el año 2005, fue desarrollada solo para utilizarse para propósitos internos a la compañía \textit{Symbian Foundation} y para mejorar la seguridad y compatibilidad de todas las versiones comprendidas entre la 6.0 y la 8.1. Además, en esta versión, se utiliza ampliamente el kernel EKA1, que alcanza su versión final.

En lo que respecta a la versión Symbian OS 9.1, ésta fue lanzada en el año 2005, centrándose en la mejora de seguridad del sistema operativo, para evitar problemas como el ocurrido en la versión 7.0. Así, se incluye un nuevo módulo que facilita la firma de aplicaciones, por lo que los desarrolladores de aplicaciones tienen que solicitar una firma digital, utilizar el nuevo API de desarrollo y firmar las aplicaciones. Posteriormente, esta firma se autentica con el programa \textit{Symbian Signed} y se permite su instalación. \\
\indent Por otra parte, esta versión introduce el soporte para la tecnología Bluetooth 1.2 y soporte para el protocolo de gestión de dispositivos especificado por la OMA (\textit{Open Mobile Alliance}), en su versión 1.1.2, que permite la configuración, actualización y gestión de fallos del dispositivo móvil. Dicha versión se utiliza en las series \textbf{Series 60 3rd Edition} y \textbf{UIQ 3.0}, en teléfonos como el \textit{Nokia N75} o el \textit{Sony Ericsson G700}.

La versión 9.2, lanzada en el primer trimestre del año 2006, añade el soporte para Bluetooh 2.0 y para la versión 1.2 del protocolo de gestión de la OMA, además de añadir el lenguaje vietnamita al sistema operativo. Esta versión se utiliza en las series \textbf{Series 60 3rd Edition}, \textbf{UIQ 3.1} y \textbf{UIQ 3.2}, en teléfonos como el \textit{Nokia N95}, \textit{Nokia E90} o el \textit{Motorola Z10}.

La versión 9.3, lanzada en Julio de 2006, mejora la gestión de memoria del dispositivo y añade soporte nativo para el estándar HSDPA (\textit{High-Speed Downlink Packet Access}), Wi-Fi 802.11 y para la especificación UMA (\textit{Unlicensed Mobile Access}), que permite recibir y desviar llamadas a través de Internet si el dispositivo está conectado a una red Wi-Fi. Esta versión se utiliza en las series \textbf{Series 60 3rd Edition}, \textbf{UIQ 3.1} y \textbf{UIQ 3.3}, en teléfonos como el \textit{Nokia N96}, \textit{Nokia 5730 XpressMusic} o el \textit{Sony Ericsson P5}.

En la versión 9.4 (publicada el año 2007) se permite el concepto de paginación bajo demanda, lo que permite cargar las aplicaciones un 75\% más rapido, ya que se van cargando los recursos en memoria según se necesiten. Además, se da soporte a SQL, a través del SGBD \textit{SQLite} y se lanza la primera palataforma de desarrollo para Symbian, la plataforma \textbf{Symbian$\wedge$1}.\\
\indent Esta versión se utiliza principalmente en la serie \textbf{Series 60 5th Edition}, en teléfonos como el \textit{Nokia 5800 XpressMusic}, el \textit{Nokia N97} o el \textit{Samsung i8910 Omnia HD}

En lo que respecta a la versión 9.5 de Symbian (lanzada en el año 2007), ésta incluye soporte nativo para televisión digital (DVB-H), soporte para servicios de localización y mejoras multimedia, como la orientación de imágenes según la posición del teléfono o el manejo de zoom óptico. Esta es la última versión conocida del sistema operativo Symbian.

Para terminar, cabe destacar que la \textit{Symbian Foundation} liberó la plataforma de desarrollo Symbian en el año 2009, bajo licencia \textit{Eclipse Public License (EPL}). En cuanto a cifras de mercado, a fecha del año 2009, se han comercializado más de 250 millones de dispositivos que ejecutan el sistema operativo Symbian.

\subsection{SDKs y entornos de desarrollo para Symbian}

Nokia ofrece varios kits de desarrollo (SDK) con los que se pueden crear nuevas aplicaciones para las diferentes versiones del sistema operativo Symbian OS explicadas en el apartado anterior. En la actualidad, hay disponibles SDKs para las series \textbf{Series 60 3rd Edition} y \textbf{Series 60 5th Edition} (\textbf{Symbian$\wedge$1}), y además hay una versión \emph{alpha} para la incipiente plataforma de desarrollo de código abierto \textbf{Symbian$\wedge$3}.

Cada kit de desarrollo incluye todos los recursos necesarios para crear, compilar, probar y distribuir aplicaciones para Symbian OS. Así pues, disponen de una amplia documentación, varias aplicaciones de ejemplo, diversos compiladores y emuladores, etc. Sin embargo, una herramienta que no se incluye en los SDK es el entorno de desarrollo (IDE), ya que no es imprescindible para utilizar el SDK. De todas formas, Nokia ha desarrollado varios entornos propios y plug-ins para entornos existentes que facilitan la utilización de los SDK.

De forma resumida, las características más importantes de los kits de desarrollo para Symbian OS son las siguientes:
\begin{milista}
	\item Bibliotecas y APIs para todos los lenguajes soportados por el SDK que dan acceso a la mayoría de las funciones de los dispositivos Symbian.
	\item Emuladores para diferentes dispositivos y plataformas, que permiten probar las aplicaciones sin necesidad de un dispositivo real.
	\item Herramientas para definir ubicaciones y rutas, que facilitan la depuración de aplicaciones que utilizan sistemas de localización como GPS.
	\item Depuración de aplicaciones escritas en C++ y Java, tanto sobre los emuladores que incluye el SDK como sobre dispositivos reales.
	\item Utilidades de diagnóstico para los emuladores con los que se puede monitorizar el tráfico de red y las operaciones solicitadas por los programas.
\end{milista}

Los dos principales lenguajes de programación soportados por los SDKs son \textbf{Symbian C++} y \textbf{Java}. A continuación se explican las diferencias entre cada uno de estos lenguajes, qué herramientas se necesitan para programar en ellos y cuáles son los entornos de desarrollo recomendados en cada caso.

\paragraph{Desarrollo en Symbian C++}

\textbf{Symbian C++} es el lenguaje de programación nativo para Symbian OS, sobre el que se ha construido toda la plataforma de desarrollo S60. Se trata de una versión optimizada de C++ para dispositivos móviles, que incorpora varias características con las que se intenta mejorar la eficiencia en el uso de los recursos y reducir el consumo de batería y memoria. Así pues, Symbian C++ permite controlar el ciclo de vida de los objetos, manejar cadenas de texto mediante descriptores e implementar el modelo cliente/servidor.

Aunque existen otros entornos de ejecución para el sistema operativo de Symbian que permiten ejecutar aplicaciones desarrolladas con Java y otros lenguajes, el entorno de Symbian C++ es el que dispone de la API más flexible y potente, ya que permite acceder y controlar todos los aspectos del dispositivo. Sin embargo, la curva de aprendizaje de este lenguaje de programación es considerablemente mayor que la del resto de lenguajes con los que se puede programar en Symbian, debido a su complejidad y a las diferencias existentes entre el C++ estándar y el utilizado por Symbian OS.

Los SDK de Symbian ya incluyen todas las herramientas necesarias para construir aplicaciones con Symbian C++, por lo que en general no se necesita instalar ningún paquete adicional además del SDK. Sólo si se van a utilizar funciones específicas de los dispositivos (como gestión de la red inalámbrica, captura de imágenes, gráficos 3D, etc.) podría hacer falta instalar plug-ins para el SDK que añadieran nuevas bibliotecas.

En cuanto al entorno de desarrollo, el más potente que existe para Symbian C++ actualmente es \emph{Carbide.c++}, un entorno basado en Eclipse creado por Nokia y la Symbian Foundation que incluye varios plug-ins específicos que se integran con el SDK para facilitar la creación de aplicaciones para Symbian OS. Aunque las primeras versiones de este IDE eran comerciales, desde la versión 2.0 (lanzada a finales de 2008) el entorno es gratuito y de código abierto.

\paragraph{Desarrollo en Java}

Todas las versiones del sistema operativo Symbian OS a partir de la 7.0 son compatibles con la plataforma de desarrollo \textbf{Java ME} (Java Micro Edition), una versión simplificada de Java SE (Java Standard Edition) diseñada específicamente para los dispositivos móviles y empotrados.

Los teléfonos Symbian más recientes soportan el \emph{Mobile Information Device Profile} (MIDP) 2.1, implementado sobre la \emph{Connected Limited Device Configuration} (CLDC) 1.1. Además, son compatibles con un gran número de extensiones de Java ME, lo que permite utilizar este lenguaje para, entre otras cosas: comunicarse inalámbricamente con otros dispositivos mediante Bluetooth, enviar y recibir mensajes SMS, recibir datos de un GPS integrado, mostrar contenidos multimedia, renderizar gráficos 3D, etc.

Desarrollar software con Java ME es más sencillo que con C++ debido a que Java es un lenguaje de mayor nivel y automatiza tareas como la recolección de basura, la gestión de la memoria o la seguridad. Además, se tiene la importante ventaja de que una aplicación Java ME para Symbian OS puede ser fácilmente portada a otras plataformas o dispositivos que también soporten la tecnología Java ME, algo que sería mucho más complicado si la aplicación se escribiera en Symbian C++. \\
\indent No obstante, puesto que los programas Java se ejecutan sobre una máquina virtual y no directamente sobre el sistema operativo, su ejecución es más lenta, y además existen menos bibliotecas y APIs disponibles para Java ME que para Symbian C++. En general, las funciones que ofrece Java ME son suficientes para desarrollar una aplicación 'normal', pero si se quieren aprovechar todas las características de los dispositivos Symbian se debe utilizar Symbian C++.

Para construir aplicaciones con Java ME, tanto en plataformas Symbian como en cualquier otro sistema operativo o entorno que lo soporte, se necesitan instalar los kits de desarrollo \emph{Java SE Development Kit} y \emph{Sun Java Wireless Toolkit for CLDC} (que, desde la versión 3.0, ha sido sustituido por el \emph{Java ME SDK}). Estos dos paquetes, junto con el SDK de Symbian correspondiente a la plataforma sobre la que se quiera programar, contienen todas las herramientas necesarias para desarrollar software basado en Java ME para Symbian OS.

De forma análoga al Carbide.c++, Nokia lanzó un entorno de desarrollo para crear aplicaciones Java ME destinadas a dispositivos Symbian llamado \textbf{Carbide.j}. Sin embargo, en la actualidad este entorno está descontinuado, y en su lugar se pueden utilizar conocidos entornos como \textbf{Eclipse}, \textbf{NetBeans} o el que se incluye con las versiones 3.0 y superiores del kit de desarrollo \textbf{Java ME SDK}.

\paragraph{Desarrollo en otros lenguajes}

Aunque Symbian C++ y Java son los dos lenguajes de programación más utilizados en la actualidad en el desarrollo de software para Symbian OS, en los últimos años se han añadido a los dispositivos Symbian entornos de ejecución compatibles con tecnologías como \textbf{Adobe Flash Lite}, una versión reducida de Adobe Flash; o \textbf{Qt}, un framework multiplataforma para el desarrollo de aplicaciones gráficas.
